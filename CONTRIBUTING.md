# Contributing to VAINOM

Thanks for your interest in VAINOM.

## Project status

VAINOM is currently developed and maintained by a single author
(Davide Virdis). The core codebase is not open source - only
compiled binaries are distributed. See LICENSE.md for details.

## What we accept

- **Bug reports**: open an issue with clear reproduction steps,
  OS version, hardware (CPU/GPU/RAM), and VAINOM version.
- **Feature requests**: open an issue describing the use case.
  Please search existing issues first.
- **Feedback on docs and website**: pull requests welcome for
  typo fixes and clarifications in README.md and other public
  documentation files in this repository.

## What we do not accept

- Pull requests modifying binary behavior, since the source code
  is not public.
- Requests to open-source the core. VAINOM is a commercial
  project; see LICENSE.md.

## Bug report checklist

Before opening an issue, please confirm:

- [ ] Using the latest public release from vainom.com
- [ ] Issue is reproducible (not a one-off)
- [ ] Checked existing issues for duplicates
- [ ] Included: OS, VAINOM version, model used, error logs

## Security issues

Do not open public issues for security problems. See SECURITY.md.
